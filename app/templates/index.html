Web-Based Face Authentication Server

A comprehensive guide for the professional edition.

Features

Enhanced Security

  • Random Folder Generation for Uploads (Hinders Unauthorized Access)
  • File Validation (Authorized Extensions: JPG, JPEG, PNG, BMP, GIF)
  • Secure File Deletion (Temporary Folders after Processing)

Robust Face Comparison

Utilizes DeepFace library for deep learning-based facial recognition (Accurate & Reliable Matching)

Error Handling

Graceful exception management during DeepFace operations or file processing.

Usage

Face Authentication API Route (`/compare`)

POST endpoint for face comparisons between a reference image and a live image.

Request Parameters:

  • reference_image (mandatory): The reference image file.
  • live_image (mandatory): The live image file for comparison.

Response:

JSON object with a single key: `identical` (bool) - Indicates matching faces (`True` for match, `False` otherwise).

Error Handling:

JSON error messages for invalid requests or internal server errors.

  • 400 (Bad Request): Missing fields or invalid file formats.
  • 500 (Internal Server Error): Unexpected errors during processing.

Development Stack

Built upon a robust foundation of Python libraries:

  • Flask: Lightweight web framework for creating APIs.
  • Werkzeug: WSGI utility toolkit (secure file uploads with `secure_filename`).
  • DeepFace: Deep learning library for facial recognition.
  • Secrets: Built-in module for generating secure random folder names.
  • OS: Module for file system operations.
